## 1. Install DaisyUI

- [x] 1.1 Add `daisyui@^5` as a dev dependency via `bun add -D daisyui`
- [x] 1.2 Verify the installed version is compatible with Tailwind v4 (`bun run check` passes)

## 2. Configure DaisyUI as a Tailwind Plugin

- [x] 2.1 Add `@plugin "daisyui"` to the app's CSS entrypoint (locate the file that contains `@import "tailwindcss"` and add the plugin directive after it)
- [x] 2.2 Confirm DaisyUI component classes are generated by running `bun run build` and checking the output CSS

## 3. Apply Default Theme

- [x] 3.1 Add `data-theme="light"` to the `<html>` element in `src/app.html`
- [x] 3.2 Start the dev server (`bun run dev`) and confirm the page renders with DaisyUI theme tokens (e.g., inspect `--color-primary` CSS custom property in browser DevTools)

## 4. Verify No Regressions

- [x] 4.1 Visually inspect the existing app layout to confirm Tailwind utility classes still apply correctly
- [x] 4.2 Check that `@tailwindcss/forms` and `@tailwindcss/typography` still function as expected alongside DaisyUI
- [x] 4.3 Run `bun run check` (svelte-check + TypeScript) and confirm no errors
